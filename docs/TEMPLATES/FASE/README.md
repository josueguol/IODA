# Plantillas de fase

Estas plantillas estandarizan la documentacion minima por fase:

- `README_TEMPLATE.md`
- `PLAN_EJECUCION_TEMPLATE.md`
- `ESTADO_TEMPLATE.md`
- `QA_GATE_REPORT_TEMPLATE.md`

## Uso recomendado

1. Crear carpeta de fase (ej. `011-NOMBRE_FASE/`).
2. Copiar las plantillas y renombrarlas:
   - `README.md`
   - `PLAN_EJECUCION.md`
   - `ESTADO.md`
   - `QA_GATE_REPORT.md`
3. Completar secciones antes de iniciar implementacion.
4. Actualizar `ESTADO.md` y `QA_GATE_REPORT.md` durante y al cierre.

## Audiencia

- Equipo humano (producto/arquitectura/dev/QA): para visibilidad y gobierno.
- Agentes IA: para ejecutar flujo con estructura consistente y trazable.
